# 🚀 Your Supercharged Learning Workflow Guide

Welcome to your new, fun, and productive development environment! Here's how to make the most of your installed extensions.

## 🎨 Visual & Fun Extensions

### One Dark Pro Theme
- Beautiful dark theme that's easy on the eyes
- Great syntax highlighting for React/TypeScript

### Material Icon Theme
- Colorful file and folder icons
- Makes navigating your project structure much easier

### Power Mode
- Fun particle effects when you type
- Makes coding feel more engaging!

### Color Highlight
- See actual colors for hex codes like `#ff0000`
- Perfect for Tailwind CSS color classes

### Peacock
- Change your editor window color for different projects
- Command: `Cmd+Shift+P` → "Peacock: Enter a Color"

## 📚 Learning & Documentation

### Markdown All in One
- Write notes, docs, and README files
- Use `Cmd+Shift+V` to preview markdown
- Create a `LEARNING_NOTES.md` file to track your progress

### Markdown Preview Enhanced
- Live preview with diagrams and math support
- Great for documenting your learning journey

### CodeTour
- Create guided walkthroughs of your codebase
- Perfect for learning new parts of your project
- Command: `Cmd+Shift+P` → "CodeTour: Start Recording Tour"

### Quokka.js
- Live JavaScript/TypeScript scratchpad
- See results as you type
- Create a `.quokka` file to experiment with code

## 🔌 API Testing & Development

### REST Client
- Test your Supabase APIs directly in the editor
- Create `.http` or `.rest` files to test endpoints
- Example:

```http
GET https://your-project.supabase.co/rest/v1/photos
Authorization: Bearer your-anon-key
```

### Docker
- Syntax highlighting and management for Dockerfiles
- Great if you decide to containerize your app later

## ⚛️ React/TypeScript Development

### ES7+ React Snippets
- Fast component creation
- Try typing `rfc` and press Tab for a functional component
- `rafce` for arrow function component with export

### Auto Rename Tag
- Automatically renames paired JSX tags
- Makes React development much smoother

### Tailwind CSS IntelliSense
- Autocomplete for Tailwind classes
- Hover to see color previews
- Perfect for your styling workflow

## 🧪 Testing & Quality

### Vitest Explorer
- Run tests directly from the editor
- See test results inline

### Error Lens
- Shows errors and warnings directly in your code
- No more scrolling to the bottom to see issues

### Coverage Gutters
- See test coverage directly in your code
- Run `npm run test:coverage` to see coverage

## 🎯 Quick Start Tips

### 1. Set Up Your Learning Journal
Create a `LEARNING_NOTES.md` file in your project:

```markdown
# Learning Notes - theyaz.io

## Today's Goals
- [ ] Learn about React hooks
- [ ] Understand Supabase authentication
- [ ] Practice with Tailwind CSS

## Questions & Discoveries
- How does React Query work?
- What's the difference between useEffect and useMemo?

## Code Experiments
- Tried Quokka.js for quick testing
- Used REST Client to test API endpoints
```

### 2. Use the Command Palette
Press `Cmd+Shift+P` and try these commands:
- "Format Document" - Format your code
- "Go to File" - Quick file navigation
- "Peacock: Enter a Color" - Change editor color
- "CodeTour: Start Recording Tour" - Create code tours

### 3. Keyboard Shortcuts
- `Cmd+Shift+X` - Extensions
- `Cmd+Shift+P` - Command Palette
- `Cmd+P` - Quick file open
- `Cmd+Shift+V` - Markdown preview
- `Cmd+Shift+E` - Explorer
- `Cmd+Shift+G` - Source Control

### 4. Experiment with Quokka.js
Create a file called `experiment.js` and add this at the top:

```javascript
// Quokka.js experiment
const greeting = "Hello, theyaz.io!";
console.log(greeting);

// Try some React concepts
const useState = (initial) => [initial, () => {}];
const [count, setCount] = useState(0);
console.log('Count:', count);
```

### 5. Test Your APIs
Create a file called `api-tests.http`:

```http
### Test Supabase Photos API
GET https://your-project.supabase.co/rest/v1/photos
Authorization: Bearer your-anon-key
Content-Type: application/json

### Test Authentication
POST https://your-project.supabase.co/auth/v1/signup
Content-Type: application/json

{
  "email": "test@example.com",
  "password": "password123"
}
```
